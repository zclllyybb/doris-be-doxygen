\subsubsection*{template$<$typename \mbox{\hyperlink{exp_8c_a0acb682b8260ab1c60b918599864e2e5}{T}}$>$\newline
class doris\+::vectorized\+::\+Column\+Dictionary$<$ T $>$}
For low cardinality string columns, using \mbox{\hyperlink{classdoris_1_1vectorized_1_1_column_dictionary}{Column\+Dictionary}} can reduce memory usage and improve query efficiency. For equal predicate comparisons, convert the predicate constant to encodings according to the dictionary, so that encoding comparisons are used instead of string comparisons to improve performance. For range comparison predicates, it is necessary to sort the dictionary contents, convert the encoding column, and then compare the encoding directly. If the read data page contains plain-\/encoded data pages, the dictionary columns are converted into Predicate\+Column for processing. Currently \mbox{\hyperlink{classdoris_1_1vectorized_1_1_column_dictionary}{Column\+Dictionary}} is only used for storage layer.
