\doxysubsection{详细描述}
\subsubsection*{template$<$typename Lock\+Type, typename \mbox{\hyperlink{exp_8c_a0acb682b8260ab1c60b918599864e2e5}{T}}$>$\newline
class doris\+::\+Internal\+Queue\+Base$<$ Lock\+Type, T $>$}
T must be a subclass of \mbox{\hyperlink{structdoris_1_1_internal_queue_base_1_1_node}{Internal\+Queue\+Base\+::\+Node}}. 

FIFO queue implemented as a doubly-\/linked lists with internal pointers. This is in contrast to the STL list which allocates a wrapper \mbox{\hyperlink{structdoris_1_1_internal_queue_base_1_1_node}{Node}} object around the data. Since it\textquotesingle{}s an internal queue, the list pointers are maintained in the Nodes which is memory owned by the user. The nodes cannot be deallocated while the queue has elements. The internal structure is a doubly-\/linked list. nullptr \texorpdfstring{$<$}{<}-- N1 \texorpdfstring{$<$}{<}-\/-\/\texorpdfstring{$>$}{>} N2 \texorpdfstring{$<$}{<}-\/-\/\texorpdfstring{$>$}{>} N3 -\/-\/\texorpdfstring{$>$}{>} nullptr (head) (tail)

Internal\+Queue$<$\+T$>$ instantiates a thread-\/safe queue where the queue is protected by an internal Spinlock. Internal\+List$<$\+T$>$ instantiates a list with no thread safety.

To use these data structures, the element to be added to the queue or list must subclass \+::\+Node.
