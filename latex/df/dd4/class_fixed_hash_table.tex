\doxysubsection{详细描述}
\subsubsection*{template$<$typename Key, typename Cell, typename Size, typename \mbox{\hyperlink{class_allocator}{Allocator}}$>$\newline
class Fixed\+Hash\+Table$<$ Key, Cell, Size, Allocator $>$}
Used as a lookup table for small keys such as UInt8, UInt16. It\textquotesingle{}s different than a \mbox{\hyperlink{class_hash_table}{Hash\+Table}} in that keys are not stored in the Cell buf, but inferred inside each iterator. There are a bunch of to make it faster than using \mbox{\hyperlink{class_hash_table}{Hash\+Table}}\+: a) It doesn\textquotesingle{}t have a conflict chain; b) There is no key comparison; c) The number of cycles for checking cell empty is halved; d) Memory layout is tighter, especially the Clearable variants.

NOTE\+: For Set variants this should always be better. For Map variants however, as we need to assemble the real cell inside each iterator, there might be some cases we fall short.

TODO\+: Deprecate the cell API so that end users don\textquotesingle{}t rely on the structure of cell. Instead iterator should be used for operations such as cell transfer, key updates (f.\+g. String\+Ref) and serde. This will allow Two\+Level\+Hash\+Set(\+Map) to contain different type of sets(maps).
