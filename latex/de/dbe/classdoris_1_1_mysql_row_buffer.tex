\doxysubsubsection{\texorpdfstring{open\_dynamic\_mode()}{open\_dynamic\_mode()}}
{\footnotesize\ttfamily void open\+\_\+dynamic\+\_\+mode}

Why? Because the Nested-\/\+Type\textquotesingle{}s data need pushed multiple times, but mysql protocol don\textquotesingle{}t support nested type and each push will decide a column data.

How? Dynamic mode allow we push data in a column multiple times, and allow recursive push. We will think that the length of the next column is uncertain when open dynamic mode, so we will set the flag to 254(longest flag) and skip 8 bytes which used for record length, then compute the actual data length when close dynamic mode. In a recursive call(special for nested type), the mode will open multiple times, but the data is actually written in one column, so we only need to deal it at the beginning and at the end.

the code\+: mrb.\+push\+\_\+tinyint(5); mrb.\+push\+\_\+smallint(120); mrb.\+push\+\_\+int(-\/30000);

In normal mode, the buffer contains three column\+: 1-\/\textquotesingle{}5\textquotesingle{}-\/3-\/\textquotesingle{}120\textquotesingle{}-\/6-\/\textquotesingle{}-\/30000\textquotesingle{}

Same code in dynamic mode, the buffer contains a column\+: 254-\/48-\/\textquotesingle{}5\textquotesingle{}-\/\textquotesingle{}120\textquotesingle{}-\/\textquotesingle{}-\/30000\textquotesingle{}
