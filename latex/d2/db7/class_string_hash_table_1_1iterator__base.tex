A hack for Hashed\+Dictionary.

The problem\+: std-\/like \mbox{\hyperlink{class_string_hash_table_ab09ab3509135a067cf2c07bd6fe8ae43}{find()}} returns an iterator, which has to be compared to \mbox{\hyperlink{class_string_hash_table_accf9a4bd0c34d4a5f6a7dab66ea10cdc}{end()}}. On the other hand, \mbox{\hyperlink{class_hash_table_afb35e0b83af5ce79df57f7ee53529687}{Hash\+Map\+::find()}} returns Lookup\+Result, which is compared to nullptr. Hashed\+Dictionary has to support both hash maps with the same code, hence the need for this hack.

The proper way would be to remove iterator interface from our Hash\+Map completely, change all its users to the existing internal iteration interface, and redefine \mbox{\hyperlink{class_string_hash_table_accf9a4bd0c34d4a5f6a7dab66ea10cdc}{end()}} to return Lookup\+Result for compatibility with std \mbox{\hyperlink{class_string_hash_table_ab09ab3509135a067cf2c07bd6fe8ae43}{find()}}. Unfortunately, now is not the time to do this.
