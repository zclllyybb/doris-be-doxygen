56// IndexedColumn is a column with an optional "ordinal index" and an optional "value index".
57// - "ordinal index" enables us to seek to a particular rowid within the column
58// - "value index" enables us to seek to a particular value but requires IndexedColumn to store ordered values
59//
60// IndexedColumn can be used as the building blocks for implementing other data structures. For example,
61// - a bitmap index can be represented by two indexed columns, one for the term dictionary, one for the posting lists.
62// the "dictionary" IndexedColumn contains ordered terms and a value index.
63// the "posting" IndexedColumn contains bitmap for each term and an ordinal index.
64// - a bloom filter index can be represented by one indexed column containing bloom filters with an ordinal index
65//
66// Currently IndexedColumn has the following restrictions but can be extended to solve in the future
67// 1. value can't be null
68// 2. duplicated values are not supported/tested when storing ordered values
69// TODO test with empty input
70class IndexedColumnWriter {

73 const TypeInfo* type_info, io::FileWriter* file_writer);

84private:

88
