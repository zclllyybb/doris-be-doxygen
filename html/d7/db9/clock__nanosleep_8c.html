We cannot port __syscall_cp because musl has very limited cancellation point implementation. For example, c++ destructors won't get called and exception unwinding isn't implemented. Instead, we use normal __syscall here and turn on the asynchrous cancel mode to allow cancel. This works because nanosleep doesn't contain any resource allocations or deallocations.
