25// Write a block to a local file.
26//
27// The block may be logically splitted to small sub blocks in the single file,
28// which can be read back one small block at a time by BlockSpillReader::read.
29//
30// Split to small blocks is necessary for Sort node, which need to merge multiple
31// spilled big sorted blocks into a bigger sorted block. A small block is read from each
32// spilled block file each time.
