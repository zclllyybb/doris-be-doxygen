4// This file provides a few functions for hashing strings. On x86-64
5// hardware as of early 2010, CityHash64() is much faster than
6// MurmurHash64(), and passes the quality-of-hash tests in
7// ./hasheval/hasheval_test.cc, among others, with flying colors. The
8// difference in speed can be a factor of two for strings of 50 to 64
9// bytes, and sometimes even more for cache-resident longer strings.
10//
11// CityHash128() is optimized for relatively long strings and returns
12// a 128-bit hash. For strings more than about 2000 bytes it can be
13// faster than CityHash64().
14//
15// Functions in the CityHash family are not suitable for cryptography.
16//
17// By the way, for some hash functions, given strings a and b, the hash
18// of a+b is easily derived from the hashes of a and b. This property
19// doesn't hold for any hash functions in this file.
20
