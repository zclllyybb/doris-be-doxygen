30// Used to cache a user function. Theses functions include
31// UDF(User Defined Function) and UDAF(User Defined Aggregate
32// Function), and maybe include UDTF(User Defined Table
33// Function) in future. A user defined function may be splitted
34// into several functions, for example, UDAF is splitted into
35// InitFn, MergeFn, FinalizeFn...
36// In Doris, we call UDF/UDAF/UDTF UserFunction, and we call
37// implement function Function.
38// An UserFunction have a function id, we can find library with
39// this id. When we add user function into cache, we need to
40// download from URL and check its checksum. So if we find a function
41// with id, this function library is valid. And when user wants to
42// change its implementation(URL), Doris will generate a new function
43// id.
44enum class LibType { JAR, SO };
45

61 // is returned by entry. Client must call release_entry to release
62 // cache entry if didn't need it.
