3// The core Jenkins Lookup2-based hashing routines. These are legacy hashing
4// routines and should be avoided in new code. Their implementations are dated
5// and cannot be changed due to values being recorded and breaking if not
6// preserved. New code which explicitly desires this property should use the
7// consistent hashing libraries. New code which does not explicitly desire this
8// behavior should use the generic hashing routines in hash.h.

14// ----------------------------------------------------------------------
15// Hash32StringWithSeed()
16// Hash64StringWithSeed()
17// Hash32NumWithSeed()
18// Hash64NumWithSeed()
19// These are Bob Jenkins' hash functions, one for 32 bit numbers
20// and one for 64 bit numbers. Each takes a string as input and
21// a start seed. Hashing the same string with two different seeds
22// should give two independent hash values.
23// The *Num*() functions just do a single mix, in order to
24// convert the given number into something *random*.
25//
26// Note that these methods may return any value for the given size, while
27// the corresponding HashToXX() methods avoids certain reserved values.
28// ----------------------------------------------------------------------
29
30// These slow down a lot if inlined, so do not inline them --Sanjay

32uint64 Hash64StringWithSeed(const char* s, uint32 len, uint64 c);
