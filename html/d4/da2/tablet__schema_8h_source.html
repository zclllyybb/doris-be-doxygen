243 // Merge columns that not exit in current schema, these column is dropped in current schema
244 // but they are useful in some cases. For example,
245 // 1. origin schema is ColA, ColB
246 // 2. insert values 1, 2
247 // 3. delete where ColB = 2
248 // 4. drop ColB
249 // 5. insert values 3
250 // 6. add column ColB, although it is name ColB, but it is different with previous ColB, the new ColB we name could call ColB'
251 // 7. insert value 4, 5
252 // Then the read schema should be ColA, ColB, ColB' because the delete predicate need ColB to remove related data.
253 // Because they have same name, so that the dropped column should not be added to the map, only with unique id.
