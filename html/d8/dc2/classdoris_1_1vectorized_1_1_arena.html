详细描述

Memory pool to append something. For example, short strings. Usage scenario:

  • put lot of strings inside pool, keep their addresses;
  • addresses remain valid during lifetime of pool;
  • at destruction of pool, all memory is freed;
  • memory is allocated and freed by large chunks;
  • freeing parts of data is not possible (but look at ArenaWithFreeLists if you need);

◆ alloc_continue()

char * alloc_continue ( size_t  additional_bytes,
char const *&  range_start,
size_t  start_alignment = 0 
)
inline

Begin or expand a contiguous range of memory. 'range_start' is the start of range. If nullptr, a new range is allocated. If there is no space in the current chunk to expand the range, the entire range is copied to a new, bigger memory chunk, and the value of 'range_start' is updated. If the optional 'start_alignment' is specified, the start of range is kept aligned to this value.

NOTE This method is usable only for the last allocation made on this Arena. For earlier allocations, see 'realloc' method.
