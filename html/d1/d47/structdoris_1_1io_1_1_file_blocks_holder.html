In CacheableReadBufferFromRemoteFS file segment's downloader removes file segments from FileBlocksHolder right after calling file_segment->complete(), so on destruction here remain only uncompleted file segments.

File segment pointer must be reset right after calling complete() and under the same mutex, because complete() checks for segment pointers.
